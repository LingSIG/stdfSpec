3. Aligning with the Open Annotation model

The Open Annotation data model articulates around three complementary components:

  • Target: that identifies one or several resources about which something is said
  • Body: that contains what is said about the resource
  • Annotation: that links one or several targets with 0-n bodies

4. Morphosyntactic annotation

ISO 24611 (MAF) contains a TEI based serialisation for morpho-syntactic annotations. The underlying model of ISO 24611 is based upon a two level model distinguishing between:

  • Tokens: elementary units segmented in the source textual content, and
  • Word forms: the abstract lexical units that may group together one or several tokens.
Note: A token can also be associated to more than one word forms, e.g. in French: du, combining de and le.

5. Reference annotation

Reference annotation consists in identifying refering expression in text or discourse and qualify the relation between them (e.g. anaphorical links) or with existing or putative external entities (e.g. in the case of deixis).

The work carried out within the project ISO24617-9 distinguishes to this purpose two types of annotation entities between which various types of links can hold:

  • referring expressions
  • discourse entities

8. Towards a content model for <annotationBlock>

Note: Intro: focusing on <annotationBlock>

In the case of an OA compliant model, the content model of <annotationBlock> shall be based upon three classes:

  • model.OATarget
  • model.OABody
  • model.OAAnnotation: a mandatory clas reference for this annotation profile. The class should contain <interp> by default and one should be cautious in adding extra element to avoid ambiguous content models when the added element may also appear in the other classes.
<content>  <sequence>   <classRef key="model.OABody"    minOccurs="0" maxOccurs="unbounded"/>   <classRef key="model.OAAnnotation"    minOccurs="1" maxOccurs="1"/>   <classRef key="model.OATarget"    minOccurs="0" maxOccurs="unbounded"/>  </sequence> </content>

In order to differentiate between the different encoding variants we have identified, the <annotationBlock> element must be associated to a notation attribute which can take one of the following values:

  • inSitu: non stand-off usage of annotation Block as documented in the Transcriptions of speech chapter.
  • OAModel: Open Annotation compliant model (Target, Body, Annotation)
